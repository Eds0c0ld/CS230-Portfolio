# CS230-Portfolio

## Project Overview

The client for this project was The Gaming Room. They wanted to take their existing Android game, Draw It or Lose It, and turn it into a web-based application that could run on multiple platforms. The main goal was to support multiple users at the same time while keeping the system scalable and easy to manage.

## What I Did Well

One thing I think I did well was keeping everything organized and sticking to the requirements. Breaking the system down into classes like Game, Team, and Player made it easier to understand how everything connects.

## What Helped Me

Working through the design document first made coding a lot easier. It gave me a clear idea of how everything should work before I even started writing code, so I wasn’t just guessing as I went.

## What I Would Improve

If I had to go back and change something, I would improve the Evaluation section. At first my answers were more basic, but later I realized I needed to go more in depth, especially when it came to things like cost and scalability.

## Understanding User Needs

When working through this project, I focused on making sure the system could handle multiple players, keep data consistent, and work across different platforms. It’s important to understand what the user actually needs, because if the design doesn’t match that, then the system won’t really be useful.

## My Design Approach

When designing the software, I tried to keep things simple and structured. Using object-oriented concepts like inheritance and encapsulation helped keep everything organized. Going forward, I would keep using this same approach by planning things out first before jumping into coding.

## Final Thoughts

This project helped me see how important planning and design are before writing code. It also showed me how systems need to be flexible and scalable, especially for web-based applications that support multiple users.
